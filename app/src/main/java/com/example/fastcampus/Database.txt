데이터베이스(Database)
- Database Administration (DBA) : 회사가 규모가 크면 있음
    - 회사의 규모가 작은 경우(스타트업, 중소기업) : DBA 가 아닌 back-end 개발자(server 개발자) 분들이 다룸
    - 안드로이드 개발자도 어느 정도 개념을 알고 있어야, 서버 개발자들과 의사소통을 잘 할 수 있다.

- 데이터란?
    - '디지털'화 되어 있는 모든 것 -> 컴퓨터의 저장 장치에 존재할 수 있는 것

- 데이터 저장방식
    - 데이터 저장 속도, 저장되어 있는 데이터를 다시 불러오는 속도
    - 저장 속도가 빠르면 다시 불러오는 속도가 느려진다.
    - 저장 속도가 느리면 다시 불러오는 속도가 빨라진다. (검색 엔진)
    1) 계층형 데이터베이스: '파일 트리(가지 치기) 방식'으로 데이터 저장 ex) window(탐색기), mac(파인더), android studio 도 파일 트리 구조임.
    2) 관계형 데이터베이스: 데이터를 행과 열을 가지는 표 형식으로 저장 ex) excel
        - 가장 보편적으로 사용되는 방식
        - oracle(유료, 매우 안전함, 데이터에 매우 민감한 부분)
        - postgreSQL(포스트그래SQL, 무료), SQLite(무료)
        - SQL 로 데이터베이스를 조작한다.
    3) 객체지향 데이터베이스: '객체'를 저장하는 방식. (객체: 클래스의 인스턴스)
        - 사용 빈도가 높지 않다.
    4) (key, value) 방식 -> 딕셔너리
        - key 를 통해서 value 를 찾는 방식
        - NoSql

- 데이터베이스를 조작하기 위한 언어
    - SQL -> 2. 관계형 데이터베이스 를 조작하기 위한 언어

- RDBMS (Relational Database Management System) : 관계형 데이터 관리 시스템
    - 관계형 데이터베이스를 관리하기 위한 소프트웨어
        - InteliJ(android studio를 만든 회사) 에서 만든 데이터 그립, 디비버
