SharedPreference(공유되는 기호 = 복잡한걸 저장하지 말라)
- 데이터 저장 가능 -> key-value 방식
- 휴대폰에 데이터 저장
- 복잡하게 사용하는 용도로 만들어진 것이 아님
- 데이터는 앱 밑에 있음
    - 휴대폰(안드로이드 OS 는 파일을 계층형 데이터베이스로 저장함)
    - 앱을 삭제하면 데이터도 삭제됨
        - manifest 의 <application> 태그에서 android:allowBackup="true" 를 하면 앱을 삭제해도 데이터가 사라지지 않을 수 있음
- Preference : 사용자의 기호에 해당하는 '간단한' 데이터를 저장해야함 -> 무거운 데이터를 저장하기에는 적합하지 않음
- Shared : 공유가 가능한 = 앱끼리 공유가 가능하다. ex) 카카오페이, 카카오뱅크, 카카오톡
    - 카카오페이에서 카카오톡에 저장된 사용자의 데이터를 참고함 (앱끼리 공유한다)
- CRUD : 데이터베이스의 기본 기능. Create, Read, Update, Delete 기능이 있음
- 파일에 저장하기 때문에 앱을 종료하고 다시 켜도 데이터가 사라지지 않음
- 데이터베이스의 장점
    - 앱이 종료가 되어도 데이터를 유지할 수 있음
- 사용 빈도
    - 많지 않지만, 전혀 사용을 안하진 않음.
    - 예) 로그인 기능 (로그인을 하고 앱을 껐다 켰을 때 로그인이 이미 되어있는 경우)
